칠월의 길

July · MMXXVI 세 낱장
07-08 수 ✦ glob 패턴과 pathlib 블로그 빌드 스크립트가 하위 폴더의 글을 놓치고 있었다. Path.glob('*.md')는 한 단계만 훑는다 — 재귀는 **/*.md, 그리고 그 축약형이 rglob. pythonpathlib

글 목록을 만드는 스크립트가 posts/ 바로 아래만 읽고 있었다. 원인은 glob 패턴.

  • Path('posts').glob('*.md') — 한 단계만 훑는다.
  • glob('**/*.md') — 하위 폴더까지 재귀. rglob('*.md')와 같다.
  • 반환값은 제너레이터라 순서 보장이 없다. 날짜순이 필요하면 sorted()로 감쌀 것.

덤으로 .obsidian 같은 숨김 폴더도 걸려 나온다. 점으로 시작하는 경로는 직접 걸러야 했다.

07-05 일 ✦ CSS color-mix()로 문장색 파생하기 장소마다 배경·보더·글로우 색을 손으로 만들던 걸 color-mix() 한 줄씩으로 정리했다. 문장색 하나에서 전부 파생된다. csscolor

이 교차로의 장소 문장색은 열두 개. 각각 배경·보더·글로우 변형까지 손으로 계산하면 서른여섯 개의 하드코딩이 된다.

  • 옅은 배경: color-mix(in srgb, var(--place) 8%, var(--card))
  • 반투명 보더: color-mix(in srgb, var(--place) 45%, transparent)

투명색과도 섞인다는 게 핵심 — rgba 값을 따로 만들 필요가 없다. 보간 색공간을 in oklch로 바꾸면 중간색이 덜 탁해지는데, 은은한 워시에는 오히려 srgb 쪽이 차분했다.

07-01 수 ✦ git rebase --onto, 세 인자의 정체 브랜치에서 갈라진 브랜치를 main 위로 옮길 때 쓰는 rebase --onto, 인자 세 개가 각각 무엇인지 드디어 외웠다. git

git rebase --onto main feat-a feat-b — feat-a에서 갈라져 나온 feat-b를 main 위로 옮긴다.

  • 첫째 인자: 새로 올라앉을 자리 (main)
  • 둘째 인자: 어디까지를 남의 것으로 볼지의 경계 (feat-a)
  • 셋째 인자: 실제로 옮길 브랜치 (feat-b)

"경계 이후의 커밋만 떼어 새 자리에 붙인다"라고 읽으니 더는 헷갈리지 않는다.

유월의 길

June · MMXXVI 다섯 낱장
06-26 금 ✦ structuredClone은 생각보다 많이 복사한다 JSON.parse(JSON.stringify(x))를 아직도 쓰고 있었다. structuredClone은 Date, Map, Set에 순환 참조까지 그대로 복사한다. javascript

깊은 복사가 필요할 때마다 JSON 왕복을 쓰던 습관을 버리기로 했다.

  • structuredClone(obj) — Date, Map, Set, TypedArray, 순환 참조까지 처리.
  • JSON 왕복은 Date를 문자열로 바꾸고, undefined를 조용히 지우고, 순환 참조에서 던진다.

단, 함수와 DOM 노드는 복사할 수 없다 — DataCloneError. 클래스 인스턴스는 평범한 객체가 되어 프로토타입을 잃는다.

06-19 금 ✦ SQLite, WAL 모드로 잠금 풀기 읽기가 쓰기를 막고 쓰기가 읽기를 막던 로컬 DB, PRAGMA 한 줄로 풀렸다. sqlitedb

로컬 도구의 SQLite가 가끔 database is locked을 뱉었다. 기본 저널 모드에서는 쓰기 중에 읽기가 막힌다.

  • PRAGMA journal_mode=WAL; — 쓰기는 별도 로그(-wal)에 쌓이고, 읽기는 원본을 그대로 본다.
  • 읽기와 쓰기가 서로를 기다리지 않는다. 단, 쓰기끼리는 여전히 직렬.

주의: -wal, -shm 파일이 함께 생기고, 네트워크 파일시스템 위에서는 권장되지 않는다.

06-14 일 ✦ Cache-Control: no-cache는 캐시한다 no-cache는 "캐시하지 마라"가 아니었다. "캐시하되, 쓰기 전에 서버에 물어봐라"다. 저장 자체를 막는 건 no-store. httpcache

이름이 가장 큰 함정인 헤더.

  • no-cache: 저장은 한다. 쓰기 전에 반드시 재검증(조건부 요청) — 그래서 304가 돌아온다.
  • no-store: 아예 저장하지 않는다. 민감한 응답용.
  • max-age=0, must-revalidate가 사실상 no-cache와 같은 효과.
06-09 화 ✦ uv run과 인라인 스크립트 의존성 일회용 스크립트에 가상환경 만들기가 귀찮아 미루던 일 — uv run이면 스크립트 파일 하나로 끝난다. pythonuv

스크립트 머리에 의존성을 적어두면 uv가 실행 시점에 알아서 격리 환경을 만든다.

# /// script 블록에 dependencies = ["requests"]를 적고 uv run fetch.py — 끝. uv add --script fetch.py requests로 블록을 자동 기록할 수도 있다.

PEP 723 표준이라 uv 전용 문법도 아니다.

06-03 수 ✦ 파이프라인의 종료 코드와 pipefail 중간 명령이 실패해도 파이프라인은 성공으로 끝난다 — 종료 코드는 마지막 명령의 것이니까. CI에서 조용히 삼켜진 실패의 범인. bashshell

배포 스크립트의 중간 명령이 죽었는데 CI는 초록불이었다.

  • 파이프라인의 종료 코드는 기본적으로 마지막 명령의 것.
  • set -o pipefail — 파이프 중 하나라도 실패하면 전체가 실패.
  • 스크립트 첫 줄 세트: set -euo pipefail.

여기서 길이 시작되었다. 첫 낱장 이전의 기록은 없다.
