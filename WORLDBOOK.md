# 📜 The Wanderer's Crossroads — Worldbook

> 이 문서는 블로그의 세계관·디자인·체계에 대한 유일한 진실의 원천(Single Source of Truth)이다.
> 모든 설계 결정은 이 문서를 따르며, 바뀔 경우 이 문서를 먼저 고친다.

---

## 1. 세계관 (Lore)

### 1.1 세계

세상의 모든 길이 만나는 곳에 오래된 교차로가 있다.
한 방랑자가 오랜 여행 끝에 이곳에 멈추어, 길마다 자신이 배운 것을 세우기 시작했다.
탑을 쌓고, 서고를 열고, 모닥불을 피웠다. 그렇게 교차로는 작은 마을이 되었다.

- **The Wanderer(방랑자)** = 블로그 주인. 이 세계의 유일한 상주자.
- **여행자(Traveler)** = 방문자. 교차로에 잠시 들른 손님.
- **장소(Place)** = 카테고리. 교차로에 세워진 12개의 건물/공간.
- **기록(Record)** = 글. 방랑자가 각 장소에 남기는 문서.
- **표식(Mark)** = 태그. 기록에 매달린 여정의 표식.
- **연대기(Chronicle)** = 시리즈/연재물.
- **일지(Journal)** = TIL. 장소에 속하지 않는 방랑자 개인의 수첩.

### 1.2 세계관 용어 사전 (UI 카피)

| 표준 용어 | 이 세계의 이름 | 비고 |
|---|---|---|
| 홈 | 교차로 (The Crossroads) | 지도가 펼쳐지는 곳 |
| About | 여행자의 쉼터 (The Wayfarer's Rest) | 여관. 방랑자 소개 |
| 전체 글 목록 | 서고 (All Records) | 시간 역순 아카이브 |
| TIL | 방랑자의 일지 (Wanderer's Journal) | |
| 시리즈 | 연대기 (Chronicles) | |
| 검색 | 길잡이에게 묻기 | Pagefind |
| RSS 구독 | 전서구 (Carrier Pigeon) | |
| 댓글 | 모닥불 곁 대화 | giscus |
| 초안(draft) | 봉인된 기록 | 빌드에서 제외 |
| 404 | 지도에 없는 땅 | "이 길은 아직 그려지지 않았다" |
| 다크모드 토글 | 여명 ↔ 모닥불 | 해/달 아이콘 대신 |

**⚠️ 절대 규칙**: 세계관 용어는 **화면에 보이는 UI에만** 쓴다.
`<title>`, meta description, OG, aria-label, URL 등 기계와 검색엔진이 읽는 곳은 **평범하고 정확하게** 쓴다.
(예: 페이지 제목은 "Python 데코레이터 정리 — The Wanderer's Crossroads"이지 "연금술사의 세 번째 기록"이 아니다.)

문장색은 장소 페이지의 헤더 리본, 글 카드의 밀랍 봉인 점, 태그 배지에만 **은은하게** 쓴다. 본문 안까지 물들이지 않는다.

### 2.2 장소 입간판 (플레이버 텍스트)

각 장소 페이지 상단에 한 줄씩 새겨 넣는다.

1. **기록관** — "잊힌 것은 사라진 것이 아니다. 아직 읽히지 않았을 뿐."
2. **수학자의 탑** — "한 층을 오를 때마다, 세계는 더 단순해진다."
3. **연구실** — "만물은 변한다. 이해한 자만이 그 변화를 이끈다."
4. **공학자 길드** — "우리는 마법을 믿지 않는다. 다만 동작하는 것을 믿는다."
5. **도서관** — "모든 책은 주문서다. 읽는 순간 발동된다."
6. **정원** — "서두르지 마라. 이곳에서는 생각도 계절을 따라 자란다."
7. **예배당** — "은혜는 값없이 주어지나, 결코 값싸지 않다."
8. **광장** — "세계의 소음이 이곳에 모여 목소리가 된다."
9. **집필실** — "이곳에서 태어난 세계들은 문 밖의 세계만큼 진짜다."
10. **아틀리에** — "말로 다 할 수 없어, 색이 되었다."
11. **거울** — "거울은 미래를 보여주지 않는다. 미래를 만드는 자를 비출 뿐."
12. **모닥불** — "장작은 넉넉하다. 앉아서 몸을 녹이고 가시게."
- **일지** — "완성되지 않아도 좋다. 오늘 걸은 만큼만 적는다."

### 2.3 방랑자의 칭호 (장소별 랭크)

## 3. 디자인 시스템

### 3.1 팔레트

**라이트 모드 — "새벽의 양피지"**

| 역할 | 값 | 설명 |
|---|---|---|
| 배경 | `#FAF5E9` | 크림 양피지 |
| 카드/코드 배경 | `#F3EBD8` | 한 톤 짙은 양피지 |
| 본문 텍스트 | `#3A3226` | 잉크 브라운 |
| 보조 텍스트 | `#6E6353` | 바랜 잉크 |
| 주 포인트 (링크/강조) | `#8B6914` | 고대의 황금 잉크 (전작 계승 — 크림 위에서 금이 잉크처럼) |
| 밝은 황금 (eyebrow 등) | `#A67C1A` | |
| 보조 포인트 | `#40826D` | 비리디안 (태그·인라인 코드) |
| 테두리/괘선 | `#DFD3B8` | |

**판형 원칙 (v3)**: 카드·버튼·배지는 **각진 판형** — 둥근 모서리(border-radius)를 쓰지 않는다.
필사본과 금속 장식의 세계에 둥근 위젯은 없다. 위계는 색보다 명도차와 타이포그래피로 만든다.

**다크 모드 — "칠흑의 숲, 황금 등불"** (v4 개정 — 반 단계 밝게, 황금 증량)

| 역할 | 값 | 설명 |
|---|---|---|
| 배경 | `#15211A` | 숲의 어둠 (순검정 금지, v4에서 반 단계 밝게) |
| 카드/코드 배경 | `#1D2C23` (hover `#243529`) | |
| 본문 텍스트 | `#E5E0D1` | 양피지의 기억 (따뜻한 미색) |
| 보조 텍스트 | `#9BB0A4` | 세이지 회녹색 |
| **주 포인트 (링크/강조)** | `#D4AA30` | **황금 — 등불의 빛. 라이트의 금과 이어지는 힌지** |
| 보조 포인트 (태그/인라인 코드) | `#5FA88B` | 비리디안 — 이끼의 인광 |
| 어두운 황금 (eyebrow 등) | `#B8932A` | |
| 괘선 | `#33473B` | |

- 대비는 항상 WCAG AA 이상 유지. 문장색(§2.1)도 라이트/다크 두 벌로 관리한다.
- 양피지 질감: 아주 옅은 노이즈 텍스처(CSS)만 허용. 진짜 양피지 사진 배경은 **금지** — 촌스러워진다.

**수채 워시 (v4)** — 카드 배경은 플랫 단색 대신 극히 옅은 2색 그라데이션 워시.
다크: 좌상단 비리디안 9% → 우하단 금 4% (155deg), 호버 시 13%/6%. 라이트: 장미빛(`#C0655B`) 4% → 앰버 3% (160deg).
grain 레이어가 수채 종이의 입자 역할. 워시는 "있는 듯 없는 듯"이 목표 — 더 진해지면 위계가 무너진다.
다크 모드에서 비리디안은 조연으로 확대(일지 액센트, 호버 테두리 gold 72:viridian 28 혼합, 안개의 기운) — 주연은 언제나 황금.

### 3.2 타이포그래피

| 용도 | 폰트 | 비고 |
|---|---|---|
| 영문 디스플레이 (히어로/브랜드/eyebrow) | **Cinzel** | 로마 비문 세리프 — 판타지 제목의 무게감 (`@fontsource/cinzel`) |
| 제목 (h1–h3) | **고운바탕 (Gowun Batang)** | 고서의 세리프 감성. 오픈소스라 셀프호스팅 용이 (`@fontsource/gowun-batang`) |
| 본문 | **Pretendard Variable** | 장문 가독성 최우선 (`pretendard` npm) |
| 코드 / 아스키 아트 | **나눔고딕코딩** (`@fontsource/nanum-gothic-coding`) | 한글:영문 = 2:1 정폭. 로컬에 Sarasa Mono K/D2Coding이 있으면 그것을 우선 사용 |

- 모두 woff2 서브셋으로 **셀프 호스팅** (외부 CDN 의존 금지).
- 본문: 17px, 줄간 1.8, 본문 폭 최대 68ch — 긴 산문 기준으로 잡는다.
- 픽셀 폰트(Galmuri 등)는 **사용하지 않는다**. 메인 감성은 정통 고서 판타지이며, 레트로 게임 감성과 섞지 않는다. (확정 결정)

### 3.3 장식 규칙 (고서의 문법)

- **드롭캡**: 에세이성 장소(정원, 예배당, 모닥불, 집필실)의 기록은 첫 문단 첫 글자를 2줄 높이 드롭캡으로. 기술 문서(길드, 연구실 등)에는 쓰지 않는다.
- **구분선**: `<hr>`은 선 대신 `✦ ── ✦ ── ✦` 오너먼트로 렌더링.
- **인용문**: 왼쪽에 가는 황금(라이트)/비리디안(다크) 괘선 + 카드 배경. 이탤릭 남용 금지(한글 이탤릭은 가독성이 나쁘다).
- **글 카드**: 양피지 카드 + 좌상단에 장소 문장색의 작은 밀랍 봉인 점.
- **소설(집필실) 전용**: 장 제목 아래 오너먼트, 챕터 이동 내비게이션(← 이전 화 | 목차 | 다음 화 →) 필수.
- **수학(탑) 전용**: 정리/증명/보조정리 박스 스타일. KaTeX 렌더링.
- 장식은 **페이지당 3종을 넘기지 않는다**. 고서의 품위는 절제에서 나온다.

**v4 셸 컴포넌트 (2026-07 리디자인 확정 — 스펙 원본: `docs/mockups/`)**
- **폴리오 프레임**: 뷰포트 네 모서리의 코너 플러리시(절차 생성 덩굴) + 짧은 이중 괘선 스텁이 여백 속으로 페이드아웃. 전체 괘선 프레임은 **쓰지 않는다**(답답함). 헤더를 침범하지 않고 헤더 아래부터 시작. 다크에서 금선 발광.
- **프로스트 글라스 헤더**: 60px 한 줄(나침반 문장 + 브랜드 + 내비 + 길잡이에게 묻기 + 여명↔모닥불 토글). 스크롤 진행도 `--hd-p`(0~1, smoothstep)로 **연속 적응** — 히어로 최상단 완전 투명 → 콘텐츠 도달 시 상한(배경 55% · blur 10px · 헤어라인 75%)의 반투명 유리. 꽉 찬 불투명 바는 금지.
- **히어로 (홈)**: 100dvh 풀블리드 시네마틱 — 금박 그라디언트 타이포가 스태거로 떠오른다. 라이트 하늘 = 여명 그라디언트(`#b8bfce→#d4bcbe→#e4c8a8→#ecd8ba→#faf5e9`, 붉으스름한 신비), 다크 하늘 = 밤의 여정(`#0b2438→#0e2a30→#14291c→#1f2c1b→#15211a` — 밤하늘 파랑에서 페트롤 청록, 다크 포레스트, 황금 올리브를 지나 배경으로. 여명과 평행한 색상환 여정). 서브페이지 배너(PageBanner)도 같은 하늘을 압축해 쓴다. (v3의 일러스트 장면 히어로는 `docs/legacy-design-system.md`로 은퇴.)
- **전역 파티클**: 문서 좌표에 앵커(뷰포트 고정 금지 — 스크롤하면 콘텐츠와 함께 흘러가고 아래엔 새 파티클). 화면당 다크 별 44+반딧불이 12 / 라이트 황금 먼지 44+발광 12 — **양 테마 동등 밀도·밝기**. 캔버스는 z-index상 카드·본문 아래(배경과 여백에서만 반짝인다).
- **전역 앰비언트 그라데이션**: body 고정 레이어 — 다크: 상단 등불빛 금 3%, 라이트: 여명 장미빛 3%.
- **헤드피스/테일피스**: 섹션 제목 위아래 좌우대칭 덩굴 오너먼트(절차 생성). 2안 "채색 필사본"에서 계승.
- **서브페이지 배너**: 100dvh 히어로 대신 220~400px 컴팩트 배너(여명/숲 축소 그라데이션 + eyebrow + 제목 + 헤드피스 + 플레이버). 장소 페이지는 문장 + RuneRing으로 한 단계 웅장하게.
- **캔버스 파티클(구 사양)·OrnamentDivider·RuneRing·grain 이중 레이어**는 v4에 계승 (파티클은 위 전역 사양으로 대체).
- 모든 모션은 `prefers-reduced-motion` 존중.
- UI 카피는 `docs/copy-deck.md`(방랑자의 어투)를 단일 원천으로 쓴다.

### 3.4 아이콘·이미지

- 파비콘: 나침반 별(✦, compass rose) 단색 — 사이트 마크 `emblem-crossroads`와 동일 도형.
- 장소 심볼: 자체 제작 모노라인 SVG 문장(紋章) 세트 — **`assets/emblems.svg`** 스프라이트 한 파일이 원본. 48×48 뷰박스, stroke 2.5, round cap/join, `currentColor` 상속(문장색은 CSS에서). 12장소 + 방랑자의 일지(나침반) + 교차로(컴퍼스 로즈) = 14종. 새 장소가 생기면 같은 문법으로 이 파일에 추가한다.
- OG 이미지: 양피지 배경 + 제목 + 장소 문장을 빌드 시 자동 생성(satori).
- 홈 지도: **v1은 3×4 카드 그리드**(장소 카드), v2에서 SVG 일러스트 지도로 승격. 일러스트가 준비되지 않았다고 출발을 늦추지 않는다.

## 4. 정보 구조 & 기술

### 4.1 URL 설계

```
/                          교차로 (홈, 지도)
/places/                   열두 장소 목록
/places/{place}/           장소별 기록 목록 (예: /places/engineers-guild/)
/records/{slug}/           개별 기록 — 장소에 종속시키지 않는다 ★
/journal/                  방랑자의 일지 (컴팩트 리스트)
/journal/{yyyy-mm-dd}/     일지 항목
/chronicles/{series}/      연대기(시리즈) 목차
/tags/{tag}/               표식별 목록
/rest/                     여행자의 쉼터 (about)
/records/                  서고 (전체 기록, 시간 역순)
```

★ 글 URL에 장소를 넣지 않는 이유: 나중에 글의 장소를 옮겨도 링크가 깨지지 않는다. 링크는 한번 공개되면 영원한 약속이다.

### 4.2 콘텐츠 스키마 (Astro Content Collections)

```yaml
# src/content/records/*.md
title: "Python 데코레이터, 제대로 이해하기"   # 평범하고 검색 친화적으로
description: "클로저부터 functools.wraps까지"  # 필수. OG/메타에 사용
place: engineers-guild                        # §2.1 슬러그 중 하나
date: 2026-07-09
updated: 2026-07-20        # 선택
tags: [python, decorator]  # 소문자-케밥, 자유롭게
series: python-deep-dive   # 선택. 연대기 슬러그
seriesOrder: 3             # series 있으면 필수
draft: false               # true = 봉인된 기록 (빌드 제외)
```

```yaml
# src/content/journal/2026-07-09.md — 일지는 극단적으로 가볍게
title: "glob 패턴과 pathlib"   # 선택 (없으면 날짜가 제목)
tags: [python]
```

- **집필실 규칙**: `authors-den` 기록은 `series` 필수 (소설은 연재물이므로).
- 파일 배치: `src/content/records/{place}/{slug}.md` — 폴더는 관리 편의용, URL과 무관.

## 5. 운영 헌장

### 5.1 안개 규칙 (The Fog)

- 발행된 기록이 **0편인 장소는 "안개에 싸인 땅"**으로 표시된다 — 지도에 실루엣과 이름만 보이고, 흐릿하게 처리되며 입장할 수 없다.
- 첫 기록이 발행되는 순간 안개가 걷힌다 (빌드 시 자동).
- 안개를 걷기 위해 억지로 글을 쓰지 않는다. 안개는 부끄러움이 아니라 **예고편**이다.

### 5.2 기록의 규칙

- 일지(TIL)는 3줄이어도 당당하게 발행한다. 일지의 적은 완벽주의다.
- 기록(본편)은 미래의 자신이 읽을 것을 기준으로 쓴다.
- 발행 후 수정은 자유. 큰 수정은 `updated`를 갱신한다.
- 제목·메타데이터에 세계관 용어 금지 (§1.2 절대 규칙 재확인).

### 5.3 로드맵 — 연대기의 장(章)
